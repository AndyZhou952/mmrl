# Prerequisite — GRPO & PPO (the RL machinery)

> Notation: follows [NOTATION.md](../NOTATION.md). Scoped to what carries over to diffusion RL; formulas yes, proofs no.

The Policy Gradient family in this repo transplants **GRPO** — the algorithm DeepSeek used for LLM reasoning — onto image/video generators. This primer gives you the parts that carry over and the vocabulary to read the algorithm pages.

---

## 1. Policy-gradient RL in one line

You have a policy $\pi_\theta$ that produces an output, a scalar **reward** $r$ that scores it, and you want to raise the expected reward:

$$\max_\theta \mathbb{E}_{\text{output}\sim\pi_\theta}[r]$$

The REINFORCE gradient is $\mathbb{E}[r\cdot\nabla_\theta\log\pi_\theta(\text{output})]$ — *push up the log-probability of outputs that scored well, push down the rest.* Used raw it works but is **high-variance**: a single noisy reward yanks the model around.

## 2. The baseline problem, and GRPO's answer: grade on a curve

The variance fix is to subtract a **baseline** — "was this output better *than expected*?" Classic PPO trains a second network (a **value/critic** model) to predict the expected reward. That doubles the models you train and serve.

**GRPO's trick: no examiner — grade on a curve.** For one prompt, sample a **group** of $N$ outputs, and use the group's own average as the baseline. An output's advantage is just its z-score within its group:

$$\hat A^{(i)} = \frac{r^{(i)} - \mathrm{mean}(\lbrace r^{(j)}\rbrace)}{\mathrm{std}(\lbrace r^{(j)}\rbrace) + \delta}$$

Like grading a class on a curve, you never need to know the absolute "difficulty" of the prompt — only who did better than the group. This kills the critic (≈ 50% less memory than PPO), removes prompt-difficulty as a confound, and is the single most reused ingredient in this repo (see [NOTATION.md §5](../NOTATION.md)).

## 3. PPO clipping: a leash on each update

On-policy data is generated by the *previous* weights $\theta_\text{old}$, so we re-weight by the **importance ratio**

$$\rho = \frac{\pi_\theta(\text{output})}{\pi_{\theta_\text{old}}(\text{output})}.$$

PPO then clips it so no single update strays too far — a **leash**: *don't rewrite the whole recipe because one tasting went well.*

$$\mathcal{L}_\text{clip} = -\mathbb{E}\left[\min\big(\rho\hat A, \mathrm{clip}(\rho, 1{-}\epsilon, 1{+}\epsilon)\hat A\big)\right]$$

One subtlety to file away: the leash only does its job when $\rho$ sits **near 1**. If the ratios are systematically off-centre, the clip never engages — exactly the failure [GRPO-Guard](../policy_gradient/grpo_guard.md) diagnoses when this is ported to flow models.

## 4. Why this is non-trivial for diffusion

In an LLM each token has a softmax probability, so $\rho$ is immediate. A flow model's sampler is a **deterministic ODE with no per-step probability** (see [flow_matching_basics.md](flow_matching_basics.md) §5) — so $\rho$ is undefined out of the box. Bridging that gap is what [FlowGRPO](../policy_gradient/flow_grpo.md) does: convert the ODE to an SDE so each denoising step becomes a Gaussian with a real $\log\pi_\theta$. VeRL-Omni describes the resulting update as a **"CLIP-style" loss using trajectory advantages** — i.e. the §3 objective, summed over the denoising steps of the rollout.

A second wrinkle is **credit assignment**. You only get the reward at the very end — you taste the dish only once it is plated ($x_0$) — yet the gradient flows through every stirring step (every denoising step). GRPO's simplest answer is to broadcast the single trajectory-level advantage $\hat A^{(i)}$ to all steps; much of the [academia](../academia.md) long-tail (DenseGRPO, TreeGRPO, …) is about doing better than "blame every step equally."

## 5. Vocabulary map: LLM → diffusion / VeRL-Omni

| LLM RL term | Diffusion/flow analogue (and VeRL-Omni framing) |
|---|---|
| Token sequence (the "trajectory") | **Denoising trajectory** $x_1 \to \cdots \to x_0$ in latent space (a VeRL-Omni *rollout*) |
| Per-token log-prob | Per-step **Gaussian log-prob** of the SDE denoising step |
| Reward on the full response | Reward model on the final image/video $x_0$ — often a **VLM-as-judge / OCR scorer** |
| Group of $N$ responses | Group of $N$ images generated from one prompt |
| Reference policy (KL anchor) | Frozen pre-RL model $\pi_{\theta_\text{ref}}$ |
| Value/critic network | **Removed** — replaced by the group baseline |

## 6. The two families, in one line

- **Policy Gradient** — keep this PPO-clip machinery and pay the price of making the sampler stochastic (FlowGRPO, DanceGRPO, MixGRPO, GRPO-Guard, …).
- **Direct Preference** — drop $\rho$ entirely and learn from a preference/MSE loss on final samples (DPO, AWM, DiffusionNFT, DGPO).
